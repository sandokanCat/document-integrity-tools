# Contributing

Thank you for your interest in contributing to `document-integrity-tools`! 

## Philosophy & Code of Conduct

This project is governed by a **Code of Conduct based on Meritocracy and Technical Excellence**. Please read it before contributing. We value your code, not your identity.

## Steps to convert

1. **Fork** the repository.
2. **Create a Branch** for your feature or fix (`git checkout -b feature/new-feature`).
3. **Make your changes** following the code style (see [`AGENTS.md`](./AGENTS.md) or existing files).
4. **Verify**: Ensure `check_docs.sh` passes all tests if you modify logic.
5. **Commit**: Write clear and concise messages.
6. **Push** your branch (`git push origin feature/new-feature`).
7. **Pull Request**: Open a PR to the `main` branch.

## Style Guidelines

* **Logs**: Use verbs (`[CREATED]`, `[VERIFIED]`) and consistent colors (Green/Red/Yellow for full lines, Cyan for paths).
* **Language**: Comments and primary documentation in technical English or neutral Spanish.
* **Linting**: Ensure bash scripts are robust (`shellcheck` recommended).
